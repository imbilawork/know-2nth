Pilot spec · build-ready · 2026-05-16

Document agent pilot — GCP + Cloudflare + Gemma 4 in country.

A 2–3 week pilot proving the post-Documentum + agent-readiness story on a real workload. A PDF is dropped at a Cloudflare URL; it's OCR'd, classified, summarised, and indexed; questions can be asked of the resulting corpus in natural language. Nothing leaves South Africa — a single GCP Compute Engine VM in africa-south1 with an NVIDIA L4 runs Gemma 4 31B via Ollama + LiteParse for OCR; Cloudflare R2 + Workers + Vectorize handle the public surface, storage, and search. Steady-state cost: ~R3,250 / month at moderate load.

Build-ready GCP · africa-south1 Cloudflare R2 + Workers + Vectorize Gemma 4 31B (Ollama) ~R3,250 / month
01 · Scope

What's in and out of the pilot.

✓ In scope

  • Upload a PDF → OCR → classify + summarise → store + index
  • Natural-language Q&A across the indexed corpus
  • Single GCP VM (L4 GPU) running Gemma 4 31B + LiteParse
  • Cloudflare R2 + Workers + Vectorize as the public surface
  • Bilingual SA test set (English + Afrikaans / Zulu invoices or letters)

× Out of scope

  • Full ECM (Alfresco / Mayan) — defer; pilot uses R2 + Vectorize as the storage + search layer
  • Multi-region / HA — single-VM, single-region
  • Customer-facing auth — shared API key, not real user accounts
  • Production audit / records retention — defer to the ECM phase
02 · Prerequisites

Four things to have in place before kickoff.

  1. GCP project with billing enabled and owner access to enable APIs (Compute Engine; Vertex AI optional for later).
  2. Cloudflare account with R2 enabled (Workers free tier covers the pilot).
  3. Budget envelope agreed before going live — ~R3,500 / month as a soft ceiling.
  4. L4 GPU quota in africa-south1. Check Compute Engine → Quotas for "GPUs (all regions)" and "NVIDIA L4 GPUs". Request raise if not granted — typically 1–3 working days.

File the GPU quota request first

It's the longest lead-time item. Everything else can move in parallel while waiting.

03 · Architecture

Three planes, one closed loop.

The pilot has three operational planes: Cloudflare for the public surface and the search index, GCP for compute and Gemma 4 inference, and a private Cloudflare Tunnel between them so the GCP VM never has a public IP. Why Cloudflare Tunnel between Cloudflare and GCP

The GCP VM stays private — no public IP, no firewall surface to defend. The tunnel makes the bridge endpoint reachable only from Cloudflare's network, and Cloudflare's authentication and rate-limiting protect it. This is the SA-banking-friendly topology: zero attack surface on the inference VM.

The bridge script (~/bridge.py) — FastAPI exposes one endpoint that runs OCR → Gemma 4 classification → returns JSON: 

# ~/bridge.py — 50 lines
from fastapi import FastAPI, UploadFile
from llama_parse import LlamaParse
import requests, json

app = FastAPI()
parser = LlamaParse(result_type="markdown")

@app.post("/process")
async def process(file: UploadFile):
    pdf_bytes = await file.read()
    open("/tmp/in.pdf", "wb").write(pdf_bytes)
    docs = parser.load_data("/tmp/in.pdf")
    text = "\n\n".join(d.text for d in docs)

    prompt = f"""Classify the following document and extract key fields.
Return JSON only: {{"type": "...", "summary": "...", "fields": {{...}}}}

DOCUMENT:
{text[:8000]}"""

    r = requests.post("http://localhost:11434/api/generate",
                      json={"model": "gemma4:31b", "prompt": prompt, "stream": False,
                            "format": "json"})
    return {"text": text, "analysis": json.loads(r.json()["response"])}

The Worker (src/index.ts) — two endpoints: /upload ingests a PDF; /ask answers questions over the indexed corpus: 

// src/index.ts
export default {
  async fetch(req: Request, env: Env) {
    const url = new URL(req.url);

    // POST /upload — multipart PDF, returns doc id
    if (url.pathname === "/upload" && req.method === "POST") {
      const form = await req.formData();
      const file = form.get("file") as File;
      const id = crypto.randomUUID();

      // 1. Store PDF in R2
      await env.DOCS.put(`pdf/${id}.pdf`, file.stream());

      // 2. Forward to GCP for processing
      const fd = new FormData();
      fd.append("file", file);
      const proc = await fetch(`${env.PILOT_API}/process`, { method: "POST", body: fd });
      const result = await proc.json();

      // 3. Store extracted text + analysis
      await env.DOCS.put(`text/${id}.json`, JSON.stringify(result));

      // 4. Index for search via Workers AI embedding
      const emb = await env.AI.run("@cf/baai/bge-base-en-v1.5", { text: result.text.slice(0, 4000) });
      await env.VEC.insert([{ id, values: emb.data[0], metadata: { type: result.analysis.type } }]);

      return Response.json({ id, ...result.analysis });
    }

    // POST /ask — natural-language Q&A over the indexed corpus
    if (url.pathname === "/ask" && req.method === "POST") {
      const { query } = await req.json();
      const qemb = await env.AI.run("@cf/baai/bge-base-en-v1.5", { text: query });
      const matches = await env.VEC.query(qemb.data[0], { topK: 3 });
      const ctx = await Promise.all(matches.matches.map(m =>
        env.DOCS.get(`text/${m.id}.json`).then(o => o?.json())));

      // Ask Gemma 4 with the retrieved context — call /process or a dedicated /ask endpoint
      const r = await fetch(`${env.PILOT_API}/ask`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ query, context: ctx })
      });
      return Response.json(await r.json());
    }

    return new Response("doc-pilot", { status: 200 });
  }
};

Cost optimisation: schedule the VM to stop overnight

Cloud Scheduler + the start/stop API can cut the VM cost ~50% if the pilot is business-hours only. The model weights stay on the persistent disk — only the GPU compute charge stops. Net steady-state drops to ~R1,750 / month for a 12-hour daily window.

08 · What "done" looks like

A 5-minute demo and the numbers behind it.

After 2–3 weeks: a working pilot that can ingest a PDF, classify and summarise it via Gemma 4 in Johannesburg, and answer natural-language questions about a corpus of 100+ documents — all without data leaving SA. Demonstrable in a 5-minute screen-share.

Concrete numbers on latency, accuracy, and cost-per-document that turn into the basis for a paid SA banking / insurance pilot. The architecture extends naturally to a production deployment: swap the single VM for a GKE cluster, swap the in-pilot R2-as-storage for Alfresco or Mayan EDMS on the same VPC, add Cloudflare Access for real user auth, and the rest of the loop is unchanged.
